Next.js 성장 일대기

Next.js 1.0부터 15.0까지의 주요 변경사항과 업데이트 배경

웹 개발의 진화 과정

  • 초기 웹 개발 (1990년대)
    • 정적 HTML 페이지
    • 서버에서 직접 HTML 파일 제공
    • 동적 콘텐츠 부재
  • 서버 사이드 템플릿 엔진 (2000년대 초)
    • JSP, PHP, ASP 등 등장
    • 서버에서 HTML 템플릿과 데이터 결합
    • 동적 콘텐츠 생성 가능
    • 페이지 전체 새로고침 필요

AJAX의 혁명 (2005년)

  • Asynchronous JavaScript and XML
    • 비동기 데이터 교환 가능
    • 페이지 새로고침 없이 데이터 업데이트
    • 더 나은 사용자 경험 제공
  • 주요 변화
    • 프론트엔드와 백엔드의 명확한 분리
    • 데이터 교환 레이어와 프레젠테이션 레이어 분리
    • 웹 애플리케이션의 상호작용성 향상

CSR 시대의 도래 (2010년대 초)

  • React, Angular, Vue 등 SPA 프레임워크 등장
  • 주요 특징
    • 클라이언트 사이드에서 모든 렌더링 수행
    • 초기 로드 후 빠른 페이지 전환
    • 리치한 사용자 인터페이스 구현 가능
  • 장점
    • 부드러운 사용자 경험
    • 서버 부하 감소
    • 오프라인 기능 구현 가능

CSR의 한계점

  • 성능 문제
    • 초기 로딩 시간 증가
      • 큰 자바스크립트 번들 크기
      • First Contentful Paint (FCP) 지연
      • Time to Interactive (TTI) 지연
    • 네트워크 의존성
      • 느린 네트워크에서의 성능 저하
      • 모바일 사용자 경험 저하
  • SEO 문제
    • 검색 엔진 크롤링 어려움
      • 초기 HTML이 비어있음
      • JavaScript 실행 필요
      • 크롤링 지연
    • 소셜 미디어 공유 문제
      • 메타 태그 동적 생성
      • 미리보기 이미지 문제

SSR의 부활 (2010년대 중반)

  • 서버 사이드 렌더링의 필요성 재발견
  • 주요 동기
    • SEO 최적화 필요성
    • 초기 로딩 성능 개선
    • 소셜 미디어 공유 최적화
  • 새로운 도전
    • 프론트엔드 개발자의 확장된 역할
      • 서버 운영 및 관리
      • 인프라 지식 필요
      • 배포 및 스케일링 고려
    • 복잡한 설정 작업
      • 서버 설정
      • 라우팅 설정
      • 데이터 페칭 설정
      • 빌드 최적화

Next.js의 등장 (2016년)

  • React 기반의 풀스택 웹앱 개발 프레임워크
  • 주요 목적
    • 복잡한 설정의 추상화
      • 서버 설정 자동화
      • 라우팅 시스템 통합
      • 빌드 최적화 자동화
    • 개발자 경험 개선
      • Zero Configuration
      • Hot Module Replacement
      • 자동 코드 스플리팅
  • 핵심 기능
    • 서버 사이드 렌더링
    • 정적 사이트 생성
    • API 라우트
    • 이미지 최적화
    • 국제화 지원
